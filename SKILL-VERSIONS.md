# Versiones de skills

## Convención de versionamiento

- Cada skill tiene su propia versión independiente en el frontmatter de `SKILL.md`
- Las versiones siguen semver simplificado: `MAJOR.MINOR` (ej. `0.1`, `0.2`, `1.0`)
- Cada versión se marca con un git tag: `skill/{skill-name}@{version}` (ej. `skill/ios-code-review@0.1`)
- `MINOR` sube cuando cambian instrucciones o configuración del skill
- `MAJOR` sube cuando hay cambios incompatibles (ej. cambio de schema, rename de skill)
